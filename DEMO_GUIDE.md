## ❓ 常见问题准备

### Q1: "既然是匿名，如何防止双投？"
**答案：**
```
"我们使用 Nullifier 机制。每个地址对每个提案生成唯一的
Nullifier（哈希值），提交时检查是否已使用。即使用户
有多个地址，我们也限制每个 Merkle Tree 里的地址只能
投一次。"
```

### Q2: "Merkle Tree 如何更新？"
**答案：**
```
"Merkle Root 由合约管理员更新。当有新成员加入或离开时，
重新生成 Merkle Tree 并调用 updateMerkleRoot() 函数。
这个过程是透明的，所有人都可以验证。"
```

### Q3: "为什么不直接用 ZK-SNARKs？"
**答案：**
```
"好问题！目前的 Merkle Tree 方案是 MVP 版本，开发速度快、
Gas 费低。未来我们会集成真正的零知识证明，进一步强化
隐私性。这也是我们的 Roadmap 之一。"
```

### Q4: "如果管理员作恶怎么办？"
**答案：**
```
"合约通过 Timelock 和多签机制保护。Merkle Root 更新需要
多签批准，给予社区足够时间响应。未来也会实现去中心化
的身份管理系统。"
```

### Q5: "相比其他方案有什么优势？"
**答案：**
```
"相比 Aragon 等 DAO 平台：
• 专注隐私保护
• Gas 费降低 99%
• 更轻量级

相比 Snapshot：
• 链上结算，更安全
• 真正的匿名，不依赖中心化服务器"
```
